# Match Factory 研究视图 — 详细说明（给「几乎零基础」读者）

如果你已经能打开 `http://localhost:3000`，但**不知道每一页在说什么、数字从哪来**，按本文从上到下读即可。不必一次读完，可当「说明书」反复查。

**与界面对照**：各功能页（概览、物品表、关卡索引、专题分析等）的**每个大模块旁**有灰色「说明」框，文案在仓库 `src/content/inlineHelp.zh.ts`，可与本文同步维护。

**在浏览器里直接阅读（推荐）**：启动站点后打开 **`/guide`**（顶部导航「使用说明」），内容与本文档同步。

---

## 1. 先搞清：这个网站**不是**游戏本体

| 是什么 | 不是什么 |
|--------|----------|
| 一个 **Next.js 小网站**，用浏览器展示你从手机里解出来的**数据文件** | 不能在里面玩 Match Factory |
| 数据来自你电脑上的**解压分包目录**（CSV、索引、字符串等） | 不会自动连官方服务器拉「真实运营配置」 |

你可以把它理解成：**给策划/研究用的「只读资料库 + 图表看板」**。

---

## 2. 三个层次：你真正在操作的东西

```
[ 游戏安装包 / 分包 ZIP ]
        ↓ 解压到某个文件夹（你本机路径）
[ 一堆文件：.lvl、.csv、.bundle、libil2cpp.so … ]
        ↓ 在项目里运行 npm run ingest（Python 脚本扫描、解析）
[ public/data/*.json 和 research.db ]
        ↓ 运行 npm run dev，浏览器读这些 JSON
[ 你看到的网页 ]
```

- **解压目录**：由环境变量 `MATCH_FACTORY_ROOT` 指定；不设置时，脚本里有一个默认路径（仅作者机器示例，**你一般要改成自己的路径**）。
- **`npm run ingest`**：把解压目录里的东西**读一遍**，生成网站要用的 JSON/SQLite。
- **`npm run dev`**：启动本地网站；网站**只读** `public/data/` 下的结果，不会去改你的游戏文件。

---

## 3. 两条命令：你一定要分清

### `npm run ingest`（完整管线）

- **作用**：从包体重新扫库存、解析 CSV、建关卡索引、抽 SDK 字符串、尝试远程 catalog（可能被 403）等，**一整套**。
- **什么时候用**：第一次搭建、换了游戏版本、换了解压路径、你发现很多页面数据明显旧了或空了。

### `npm run ingest:topic`（只刷新「专题分析」那一块）

- **作用**：只重新跑 `scripts/build_topic_analysis.py`，生成 **`public/data/topic_analysis.json`**（专题页用的聚合数据，含热力图等）。
- **什么时候用**：你已经完整 ingest 过，只是改了聚合逻辑或想**快速刷新**专题 JSON，不想等整条管线。

## 4. 打开网站后：顶部导航每一页是干什么的

下面按「你点导航时心里该想什么」来写。

### 概览 `/`

- **是什么**：整个 Unity 资源包有多大、多少文件、`.lvl` 在哪些顶层文件夹里、扩展名 Top 列表。
- **给谁用**：想 10 秒钟判断「包体我放对没有、资源量级多大」。

### 物品表 `/items`

- **是什么**：`items.csv` 解析后的表格（名字、分类、形状、颜色等）。
- **给谁用**：当「元素词典」查：游戏里有哪些道具类型、策划表字段长什么样。

### 关卡模板 `/templates`

- **是什么**：`level_templates_normal.csv` / `ease.csv` 的表格。**注意**：原始 CSV **没有表头**，网站里会显示成 `col_0`、`col_1`…  
- **给谁用**：看模板行和列长什么样；**具体每一列在游戏里代表什么**，要结合 Il2Cpp / 策划工具，本站**不保证列语义**。

### 关卡索引 `/levels`

- **是什么**：扫描到的每一个 `.lvl` 文件一条记录：路径、关卡 ID、变体号、文件大小等。
- **给谁用**：查「某一关有哪些变体文件」「文件大不大」；**不是**关卡内棋盘内容编辑器。

### 资源分包 `/bundles`

- **是什么**：Addressables / catalog 相关线索、bundle 扫描摘要（取决于 ingest 与 UnityPy 是否可用）。
- **给谁用**：研究资源是怎么分包、下载地址长什么样；偏工程向。

### 计费与广告 `/sdk`

- **是什么**：从 DEX、`libil2cpp` 里**筛出来的字符串**，和计费、广告 SDK 相关。
- **重要**：这是**客户端里出现的文字线索**，**不是** Google Play 商品列表、**不是** 真实价格、**不是** 服务端下发的完整商业化配置。

### .lvl 说明 `/lvl`

- **是什么**：二进制格式说明、Protobuf 抽样、`decode_raw` 示例等。
- **给谁用**：想理解「关卡文件为什么是二进制、字段大概怎么排」时读。

### Il2Cpp `/il2cpp`

- **是什么**：Il2CppDumper 导出后的**类名、字符串字面量**等摘要（若你跑过 `ingest:il2cpp` 相关步骤）。
- **给谁用**：把「游戏里的类名 / 字符串」和策划表、关卡格式对照。

### 数据研究 `/research`

- **是什么**：SQLite（`research.db`）和若干研究用 JSON 的入口说明。
- **给谁用**：想自己写 SQL 或深入查表结构时。

### 专题分析 `/analysis`（你最关心的「一屏看懂」）

- **是什么**：把**关卡、模板数值、物品、计费、广告**五块，各自做成 **KPI + 图表 + 表格 + 热力图（关卡 Tab）**。
- **下面第 5 节专门逐 Tab 讲**。

## 5. 专题分析页 `/analysis` — 每个 Tab 详细解释

进入后你会先看到 **四块 KPI 卡片**，再是一排 **Tab 按钮**。

### KPI 卡片（最上面四格）

| 卡片文字 | 大致含义 |
|----------|----------|
| 关卡 .lvl 文件 | `levels_index.json` 里有多少条**文件记录**（同一关可能在两个文件夹各有一条，所以会比「关卡数」多） |
| 单文件体积 | 这些 `.lvl` 文件**最小/最大字节数**，帮助判断体量 |
| 物品词条 | `items.csv` 有多少行 |
| 模板行数 N / E | normal / ease 两套模板表各有多少行 |

### Tab：关卡

- **目录分布条形图**：文件落在 `Levels` 还是 `DLCFallbackLevels` 等多一些。  
- **变体编号分布**：`_00`、`_01`… 对应 variant 字段；通常 `_00` 偏「基底」、`_01+` 偏关卡数据（详见 `.lvl` 说明页与 schema）。  
- **体积直方图**：文件大小集中在哪个区间。  
- **关卡 ID 段 × 变体热力图**：横轴变体、纵轴「关卡 ID 分成 20 段」；格子里的数字是**索引行数**（可能同一关两套目录各占一行）。颜色越深越多。

### Tab：关卡数值

- **对照表**：对 `col_*` 里能识别为整数的列，算 **min / median / max**（抽样），**normal 与 ease 两列并排**，方便对比难度相关列是否不同。  
- **「原始 JSON」折叠区**：给已经懂 JSON 的人调试用；不懂可以不看。

### Tab：物品

- **Category1 / Category2 / Shape 条形图**：物品表里这些维度各有多少条；用来理解「策划把物品怎么分类」。

### Tab：计费点

- **左侧**：每个「关键字分桶」里**有多少条**字符串。  
- **右侧**：分桶里的**原文**；上面有**搜索框**，可过滤。  
- **再强调**：这是**逆向线索**，不是商店 SKU 全表。

### Tab：广告

- 和计费类似：分桶 + 条数 + 可搜索原文；同样是**客户端字符串证据**，不是完整瀑布流配置。

### 最下面「二期可视化建议」

- **是什么**：仓库里 `VISUALIZATION_DESIGN.md` 同步过来的**将来还可以做什么**的列表，不是 bug。

### 最底下「数据版本 · 脚本名」

- **是什么**：`topic_analysis.json` 里的 `version` 和 `generatedBy`，用来确认你看到的是不是刚跑出来的文件。

## 7. 你「能信什么」和「不要指望什么」

**可以当证据用的：**

- 包体里**真实存在**的文件列表、大小、路径。  
- CSV 里**真实存在**的行与列值。  
- 客户端里**真实出现**的类名、字符串（在对应工具链跑通的前提下）。

**不要指望本站直接给出的：**

- 官方未公开的**完整数值策划表**（很多在服务器或加密 bundle）。  
- **准确列语义**（`col_3` 到底是什么）——需要对照代码或策划工具。  
- **真实 IAP 价格、SKU 与服务器 offer 一一对应**——当前数据源不支持。

## 8. 常见问题（FAQ）

### Q：页面一片空白或专题页提示加载失败？

1. 是否执行过 `npm run ingest`？  
2. `public/data/topic_analysis.json` 是否存在？没有则先 `npm run ingest` 或 `npm run ingest:topic`（后者需要其它 JSON 已存在）。  
3. 浏览器强制刷新（Ctrl/Cmd + Shift + R）。

### Q：为什么关卡数和我心里想的「几千关」对不上？

- 索引统计的是 **`.lvl` 文件条数**（含多文件夹、多变体），不是「玩家看到的关卡序号 1…N」的简单计数。

### Q：我能改网页里的数据吗？

- 可以改 `public/data/*.json` **做实验**，但下次 `npm run ingest` 可能被覆盖。长期改法应改 Python 脚本再重新 ingest。

### Q：端口不是 3000 怎么办？

- 若 3000 被占用，Next 会提示用 3001 等；以终端里 **`Local: http://…`** 为准。
